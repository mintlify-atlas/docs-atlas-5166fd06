## Why monaco-vim?

If you're building a web-based code editor using Monaco Editor and your users love Vim, monaco-vim is the perfect solution:

- **Familiar Editing**: Users can leverage their existing Vim muscle memory
- **Productivity Boost**: Vim's modal editing provides efficient text manipulation
- **Battle-Tested**: Built on CodeMirror's mature Vim implementation
- **Active Maintenance**: Regularly updated to work with latest Monaco Editor versions
- **Lightweight**: Minimal overhead while providing comprehensive Vim functionality

<Note>
monaco-vim requires Monaco Editor as a peer dependency. It works with modern bundlers (webpack, browserify, Vite) and AMD module loading.
</Note>

## How It Works

monaco-vim acts as a bridge between CodeMirror's Vim keybindings and Monaco Editor's API. When you initialize Vim mode:

1. The adapter attaches to your Monaco Editor instance
2. All keyboard input is intercepted and processed through Vim's state machine
3. Commands are translated into Monaco Editor operations
4. The status bar displays real-time feedback on mode and key presses

<Info>
The implementation is based on CodeMirror's `keymap_vim.js`, ensuring authentic Vim behavior and compatibility.
</Info>

## Browser Compatibility

monaco-vim works in all modern browsers that support Monaco Editor. It requires ES2015+ features and works best with:

- Chrome/Edge 90+
- Firefox 88+
- Safari 14+

<Warning>
Global script inclusion is not supported due to internal Monaco Editor dependencies. You must use a bundler (webpack, Vite, etc.) or AMD module loading.
</Warning>
